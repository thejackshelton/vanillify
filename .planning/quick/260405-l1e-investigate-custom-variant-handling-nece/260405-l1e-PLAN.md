<objective>
Remove `customVariants` option and `CustomVariantsOption` type, rename `themeCss` to `css` in `ConvertOptions`. Merge the two separate CSS string params (`customVariantsCss`, `themeCss`) into a single `css` param throughout the pipeline. Drop `variantsRecordToCss()` helper -- users write raw CSS.

Purpose: Simplify the public API surface. Both customVariants (CSS string form) and themeCss are just CSS passed to Tailwind's compile(). No reason for separate options.
Output: Unified `css` option in ConvertOptions, single `css` param in pipeline functions, updated CLI and tests.
</objective>
