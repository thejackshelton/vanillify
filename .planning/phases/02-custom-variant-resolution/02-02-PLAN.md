## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User options -> generator | customVariants flows from user code into UnoCSS createGenerator config |
| UnoCSS output -> rewriter | Generated CSS with custom variant selectors is parsed and rewritten |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Denial of Service | getGenerator cache | mitigate | Cache keyed by sorted variant names prevents unbounded cache growth from identical variant sets in different order. Cache is in-memory only (build-time tool, not long-running server). |
| T-02-05 | Tampering | generator variants config | accept | Custom variants are passed directly to UnoCSS's createGenerator which handles its own validation. Invalid variants produce no CSS output rather than corrupted output. Build-time library with no persistence. |
| T-02-06 | Information Disclosure | generator error messages | accept | UnoCSS error messages may contain variant names/selectors. Acceptable for a build-time library -- no sensitive data in variant definitions. |
</threat_model>
