Create `src/cli.ts` as a thin CLI wrapper. The file MUST:

1. Start with `#!/usr/bin/env node` shebang on line 1
2. Import ONLY from these modules:
   - `citty` (defineCommand, runMain)
   - `consola` (consola)
   - `tinyglobby` (glob)
   - `pathe` (resolve, dirname, basename, join)
   - `node:fs/promises` (readFile, writeFile, mkdir)
   - `./index` (convert) -- the ONLY import from vanillify internals
3. Use citty's `defineCommand` pattern with:
   - meta: name "vanillify", version from package.json or hardcoded "0.0.1", description "Convert Tailwind CSS classes to vanilla CSS"
   - args.patterns: type "positional", required true, description "File paths or glob patterns"
   - args.outDir: type "string", alias "o", description "Output directory (default: alongside input files)"
   - args.customVariants: type "string", alias "c", description "Path to CSS file with @custom-variant definitions"
4. In the `run` handler:
   - Expand globs with `glob([args.patterns].flat() as string[])` to handle citty positional arg behavior (Pitfall 1 from research)
   - If no files matched, log warning with consola.warn and set `process.exitCode = 1`, return
   - For each matched file:
     a. Read file with `readFile(resolve(file), 'utf-8')`
     b. If `args.customVariants` is provided, read that CSS file too
     c. Call `await convert(source, file, options)` -- delegate ALL conversion to the API
     d. Determine output dir: `args.outDir ? resolve(args.outDir) : dirname(resolve(file))`
     e. Derive output name: `basename(file).replace(/\.(tsx?|jsx?)$/, '')`
     f. Write `${name}.vanilla.css` and `${name}.vanilla.tsx` to output dir
     g. Log any warnings from result.warnings using consola.warn
   - After loop: `consola.success(\`Processed ${files.length} file(s)\`)`
5. Call `runMain(main)` at module level

CRITICAL: Zero conversion logic in this file. No class extraction, no CSS generation, no AST work. Only file I/O + glob + arg parsing + calling convert().
